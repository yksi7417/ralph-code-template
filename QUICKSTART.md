# Quick Start Guide

## What is This?

A **project template** for the **Ralph Wiggum** AI coding workflow, optimized for a **Planner-Executor** model where:

- **Planner** (Claude Code or similar) creates detailed implementation plans
- **Executor** (Local LLM like Qwen3-Coder) implements the code

## Prerequisites

1. **Git** installed
2. **One of these AI tools** configured:
   - Claude Code CLI (`claude`)
   - OpenCode CLI (`opencode`)
   - Aider (`aider`)
   - Or similar
3. **Local LLM server** (for executor mode):
   - Your Tailscale-accessible server: `https://fedora-manor.tail356fe.ts.net/`
   - Or: Ollama, LM Studio, vLLM, etc.

## Setup Steps

### 1. Copy Template to New Project

```bash
# Create new project from template
cp -r ralph-code-template my-new-project
cd my-new-project

# Initialize fresh git history
rm -rf .git
git init
git add .
git commit -m "Initial project from Ralph Wiggum template"
```

### 2. Configure for Your Project

Edit these files:

**CLAUDE.md** - Update:
- Project description
- Build commands for your language
- Local LLM URL if different

**AGENTS.md** - Update:
- Technology stack
- Build/test/run commands
- Project structure

**config.yaml** - Update:
- Planner model preference
- Executor API base URL
- Loop settings

### 3. Write Specifications

Create spec files in `specs/`:

```bash
# Copy templates and edit
cp specs/architecture-template.md specs/architecture.md
cp specs/domain-model-template.md specs/domain-model.md
cp specs/testing-strategy-template.md specs/testing-strategy.md

# Edit each file with your project's details
```

### 4. Generate Implementation Plan (Planner Mode)

```bash
# Run the planner (uses Claude or smart model)
./loop.sh plan
# or on Windows:
loop.bat plan

# This will:
# 1. Read your specs
# 2. Generate IMPLEMENTATION_PLAN.md
# 3. Create detailed, atomic tasks with code examples
```

Review the generated `IMPLEMENTATION_PLAN.md` before proceeding.

### 5. Start Building (Executor Mode)

```bash
# Run the executor (uses local LLM)
./loop.sh build
# or on Windows:
loop.bat build

# This will loop:
# 1. Pick a task from the plan
# 2. Implement it (tests first)
# 3. Validate (run tests)
# 4. Commit and continue
```

### 6. Monitor Progress

```bash
# Check what's been done
cat IMPLEMENTATION_PLAN.md | grep "\[x\]"

# Check recent commits
git log --oneline -10

# Check learnings
cat AGENTS.md | tail -50
```

## File Structure

After setup, your project should look like:

```
my-project/
├── CLAUDE.md                 # Claude Code guidance
├── AGENTS.md                 # Operational guide (how to build)
├── IMPLEMENTATION_PLAN.md    # Task list (generated by planner)
├── PROMPT_plan.md            # Instructions for planner
├── PROMPT_build.md           # Instructions for executor
├── config.yaml               # Configuration
├── loop.sh                   # Loop script (Linux/Mac/WSL)
├── loop.bat                  # Loop script (Windows)
├── QUICKSTART.md             # This file
├── specs/                    # Specifications
│   ├── README.md
│   ├── architecture.md       # Your architecture spec
│   ├── domain-model.md       # Your domain model spec
│   └── testing-strategy.md   # Your testing strategy
├── src/                      # Source code (created during build)
└── tests/                    # Tests (created during build)
```

## Tips for Success

### For the Planner (Claude)

1. **Be extremely explicit** - The executor has limited reasoning
2. **Include complete code** - Don't assume the executor can infer
3. **One task per iteration** - Keep tasks atomic
4. **Test first always** - Write test code before implementation code

### For the Executor (Local LLM)

1. **Follow the plan exactly** - Copy code from the plan
2. **Validate every change** - Run tests before committing
3. **Document issues** - Update AGENTS.md when stuck
4. **Keep it simple** - Don't add features not in the plan

### General

1. **Start small** - Begin with a simple project to test the workflow
2. **Review generated plans** - Check IMPLEMENTATION_PLAN.md before building
3. **Iterate on specs** - Refine specs based on what works
4. **Trust the process** - Ralph is persistent, not smart

## Troubleshooting

### Executor Gets Stuck

1. Stop the loop (Ctrl+C)
2. Check AGENTS.md for recorded issues
3. Manually fix the blocking problem
4. Resume: `./loop.sh build`

### Plan Seems Wrong

1. Stop the loop
2. Re-run planning: `./loop.sh plan`
3. Or manually edit IMPLEMENTATION_PLAN.md
4. Resume building

### Tests Keep Failing

1. Check if the plan's test code is correct
2. Verify dependencies are installed
3. Check AGENTS.md for correct test command
4. Update the plan if needed
